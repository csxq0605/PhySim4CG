## 2. 单刚体 Demo：`taichi_lab1_single_body.py`

### 2.1 核心实现思路

这个文件对应基础任务 1，目标是验证单个长方体刚体的平动与转动积分是否正确。

核心实现包括：

- 使用位置 `x`、线速度 `v`、角速度 `omega` 和四元数 `q` 描述刚体状态。
- 在每个时间步中先积分外力得到线速度，再更新位置。
- 使用刚体参考惯性张量和当前姿态矩阵构造世界坐标系下的惯量张量。
- 通过四元数微分方程更新姿态，再做归一化，避免数值漂移。
- 鼠标拖拽时在局部坐标系锚点处施加外力，通过 `tau = (R r) x f` 产生力矩。

第一部分的重点是验证：

- 给定初始线速度后刚体会按预期平移。
- 给定初始角速度后刚体会按预期旋转。
- 施加偏心外力后会同时产生平动和转动。

### 2.2 交互方式

- `左鼠标指针拖动`：在离中心有偏移的位置施加外力
- `右鼠标指针拖动`：绕场景中心旋转相机
- `WASDQE`：移动相机
- `R`：重新随机初始化单刚体状态
- `SPACE`：暂停 / 继续
- `ESC`：退出

## 3. 双刚体碰撞 Demo：`taichi_lab1_two_body.py`

### 3.1 核心实现思路

这个文件对应基础任务 2，目标是构造两个长方体的保证碰撞场景，并实现冲量响应。

主要实现包括：

- 使用 SAT 对两个 OBB 进行碰撞检测。
- 由碰撞检测返回碰撞法向、穿透深度和单接触点近似。
- 在接触点计算相对速度，沿法向计算法向冲量。
- 再在切向方向估计摩擦冲量，并使用库仑摩擦上界进行裁剪。
- 最后使用位置修正将两个物体分离，减少持续穿插。

第二部分主要用于观察：

- 物体能否在相撞后正确分离。
- 不同入射方向下，边-边、点-面、面-面接触时 SAT 返回的接触法向是否稳定。
- 单接触点近似在复杂接触时的局限性。

### 3.2 交互方式

## 5. Bonus B1：多体牛顿摆探索 `taichi_lab1_bonus_b1.py`

### 5.1 任务目标

B1 要求探索多体串行碰撞中，简单的冲量法是否能够实现类似牛顿摆的“完美能量传递”，并分析动能丢失或传递顺序错误的原因。

### 5.2 实现思路

本文件构造了 5 个等质量、等尺寸长方体沿一条链排布的场景，并将第一个刚体以一定速度发射向后续刚体。

具体实现：

- 不加阻尼，恢复系数设为 `1.0`，尽量排除耗散的影响。
- 设置 4 组实验预设，控制变量分别为：
  - `substeps`
  - `solver_iters`
  - `pair_order`
- 统计每个刚体的峰值速度和峰值出现顺序。
- 峰值统计按 `substep` 采样，而不是按帧采样，避免漏掉短时速度峰值。

### 5.3 交互方式

- `TAB`：循环切换实验预设并重置
- `R`：重置当前实验
- `P`：打印当前摘要
- `右鼠标指针拖动`：旋转相机
- `WASDQE`：移动相机
- `SPACE`：暂停 / 继续
- `ESC`：退出

### 5.4 日志说明

脚本会在固定帧数40帧处自动打印摘要，核心字段含义如下：

- `final_ke_ratio`：末态总动能 / 初态总动能，用于观察是否保能。
- `transfer_order`：各刚体达到峰值速度的先后顺序，用于观察传递次序是否正确。
- `body_peak_vx`：每个刚体达到过的最大 `x` 向速度。
- `body_peak_sample` / `body_peak_time`：峰值出现的子步编号和时间。

使用命令 `python taichi_lab1_bonus_b1.py --batch`，可以在不调出窗口的情况下自动依次运行完所有的4个实验预设并打印相应的实验日志。

### 5.5 实验日志与结果

4 组典型结果如下：

| 实验 | 设置 | `final_ke_ratio` | `transfer_order` | `body_peak_vx` 现象 |
| --- | --- | ---: | --- | --- |
| `baseline_serial` | `1 substep + 1 iter` | 1.000 | `0 -> 2 -> 4` | 速度错误地跳过了 1、3 号刚体 |
| `more_iters` | `1 substep + 8 iter` | 1.000 | `0 -> 2 -> 4` | 仅增加迭代次数，结果几乎不变 |
| `more_substeps_iters` | `4 substeps + 8 iter` | 1.000 | `0 -> 1 -> 2 -> 3 -> 4` | 速度按链条逐级传播 |
| `alternating_sweeps` | `4 substeps + 8 iter + alternating` | 1.000 | `0 -> 1 -> 2 -> 3 -> 4` | 与上一组接近，顺序正确 |

关键点可以概括为：

- 粗时间步下，中间刚体会被“跳过”，例如 `0 -> 2 -> 4`。
- 细化 `substeps` 后，峰值传播顺序恢复为 `0 -> 1 -> 2 -> 3 -> 4`。
- 在本实验的理想参数设置下，总动能基本守恒，说明主要误差来自接触求解顺序和时间离散，而不是材料耗散。

### 5.6 对题目问题的回答

1. 简单串行冲量处理能否实现完美能量传递？

不能稳定实现。尤其在 `substeps` 很少时，冲量会在同一大时间步内被错误分配，导致速度传播顺序出错。

2. 如果发生“动能丢失”或“传递顺序错误”，原因是什么？

本实现中的主要原因是：

- 时间步过大，多个接触在一个大步里被粗糙地近似。
- 接触求解具有顺序依赖性，先处理哪个 pair 会影响后续 pair 的相对速度。
- 多体链式碰撞本质上需要更细时间分辨率，单次串行冲量更新很难一次性得到理想结果。

3. 哪种策略能提高准确性？

从实验结果看，增加 `solver_iters` 本身效果有限，更关键的是增加 `substeps`。也就是说，对于 B1，减小时间离散误差比单纯增加同一步内的迭代次数更有效。

## 6. Bonus B2：多体稳定堆叠探索 `taichi_lab1_bonus_b2.py`

### 6.1 任务目标

B2 要求在重力下堆叠至少 3 层长方体，并观察系统能否稳定、是否会因小抖动而坍塌，以及怎样通过数值策略提升稳定性。

### 6.2 实现思路

本文件构造了 5 层长方体竖直堆叠场景，并加入地面接触。为便于比较，设计了 4 组实验预设：

- `baseline_low_iters`
- `more_iters`
- `plane_first_forward`
- `bottom_up_plane_first`

对比变量主要包括：

- `substeps`
- `solver_iters`
- 接触求解顺序
- 是否优先处理地面接触

脚本内部记录了碰撞次数、采样数、顶层高度历史和坍塌帧等指标，自动日志重点展示 `collapse_frame`，用于比较不同策略下堆叠能保持多久不坍塌。

### 6.3 交互方式

- `TAB`：循环切换实验预设并重置
- `R`：重置当前实验
- `P`：打印当前摘要
- `右鼠标指针拖动`：旋转相机
- `WASDQE`：移动相机
- `SPACE`：暂停 / 继续
- `ESC`：退出

### 6.4 日志说明

脚本会在固定帧数200帧处自动打印摘要，核心字段为：

- `frames_run`：当前实验已经运行的帧数。
- `substep_samples`：累计子步采样数量。
- `collisions`：累计处理的碰撞次数。
- `collapse_frame`：第一次满足“堆叠明显失稳/坍塌”条件的帧编号；若为 `-1`，表示在统计窗口内未坍塌。

使用命令 `python taichi_lab1_bonus_b2.py --batch`，可以在不调出窗口的情况下自动依次运行完所有的4个实验预设并打印相应的实验日志。

### 6.5 实验日志与结果

4 组实验在 200 帧处的对比如下：

| 实验 | 设置 | `collisions` | `collapse_frame` | 结论 |
| --- | --- | ---: | ---: | --- |
| `baseline_low_iters` | `2 substeps + 2 iters + forward` | 707 | 30 | 很快坍塌，稳定性最差 |
| `more_iters` | `4 substeps + 12 iters + forward` | 28537 | 127 | 增加子步和迭代后明显更稳定 |
| `plane_first_forward` | `4 substeps + 12 iters + plane-first` | 37952 | 198 | 在当前实验中效果最好 |
| `bottom_up_plane_first` | `4 substeps + 12 iters + bottom-up` | 37904 | 198 | 与 plane-first 类似，但没有继续提升 |

### 6.6 对题目问题的回答

1. 多层堆叠能否稳定？

在简单冲量法下，稳定性有限。低迭代、低子步的设置很快就会坍塌；增加 `substeps` 与 `solver_iters` 后，可以显著延缓坍塌。

2. 是否会因为微小振动而坍塌？

会。由于该系统没有显式持久接触约束，也没有完整的堆叠稳定器，重力与接触修正会不断引入小的残余速度，最终导致缓慢失稳。

3. 哪种策略能增强稳定性？

本实验中有效的策略是：

- 增加 `substeps`
- 增加 `solver_iters`
- 在每个求解轮中优先处理地面接触

从实验结果看，`plane_first_forward` 明显优于最基础的 `baseline_low_iters`，说明地面约束优先被满足后，堆叠整体更稳定。

## 7. Bonus B3：复杂几何支持 `taichi_lab1_bonus_b3.py`

### 7.1 任务目标

B3 要求支持凸包或其他 Primitive。本次实现选择了混合几何体场景，在同一场景中加入：

- `box`
- `sphere`
- `cone`
- `convex_hull`

其中凸包几何采用四面体作为示例，圆锥使用凸多面体 mesh 近似表示。

### 7.2 实现思路

主要实现包括：

- 为不同刚体设置 `shape_type / kind` 与对应几何参数。

- box、cone、convex hull 都作为一般凸多面体处理。
- 实现 `collision_manifold(i, j)` 分派：
  - `sphere-sphere`：解析求解。
  - `hull-sphere`：对球心与三角面求最近点，或判断球心在凸体内部时使用最深平面法向。

- 场景中加入地面和 4 面墙，所有物体都受重力影响。

为了控制性能，还加入了：

- 基于包围球半径的 broadphase 早退。
- 低多边形球 mesh 和低分段圆锥 mesh。
